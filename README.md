# vite template



## Начало работы

Эта сборка **vite** содержит большую часть полезных и удобных плагинов. Среди прочего в ней есть линтеры, следящие за стилем написания **js** и **scss**. Установлен **prettier** для автоформатирования кода, так что во избежание конфликтов рекомендуется отключить плагины автоформатирования вашей **ide**.

После клонировании с целью использования данного шаблона обязательно удалить связь с репозиторием.

```
git remote remove https://gitlab.wim.agency/boilerplates/wim-vite.git
```

Убедитесь в отсутствии связи с удалённым репозиторием.

```
git remote -v
```

Установите зависимости.

```
npm i
```

## О сборке
- Для указания пути к файлам, расположенным в директории **public**, в ваших html файлах в качестве пути к файлу достаточно указать `/название_файла`.
- При добавлении изображений в директорию **img** с расширениями png или jpeg, будет сгенерирована дирректория с именем webp, в которую будут добавлены данные изображения.
- В конфигурационном файле *vite.config.js* укажите расположение вашего проекта, если собираетесь проксировать запросы на адрес */api*
- Добавленные стили в директорию *src/scss/libs/* будут игнорироваться линтером стилей
```
// Адрес для проксирования запросов
const origin = 'https://your-game-origin.ru';
```

## Команды

Запуск dev среды. Для просмотра страницы в локальной сети воспользуйтесь адресом предлагаемым **vite** после сборки, например
`➜  Network: http://172.20.10.6:5173/`.

```
npm run dev
```

Сборка build версии.

```
npm run build
```

Просмотр preview версии build. Для просмотра превью в локальной сети воспользуйтесь адресом предлагаемым **vite** после сборки, например
`➜  Network: http://172.20.10.6:4173/`.

```
npm run preview
```

Команды линтинга кода. Их использование не обязательно, так как автоматически отрабатывают в dev среде с помощью плагинов.

```
npm run lint:script
npm run lint:style
npm run fix:script
npm run fix:style
```

## Предложения

Предложения по модификации сборки вносятся с помощью запроса на слияния. Далее обсуждаются с командой. Если вы хотите предложить дополнить сборку, то:

1. Склонируйте репозиторий.

```
git clone https://gitlab.wim.agency/boilerplates/wim-vite.git
```

2. Перейдите в каталог репозитория.

```
cd wim-vite
```

3. Создайте новую ветку для изменений и перейдите на неё.

```
git checkout -b new-feature
```

4. Далее внесите изменения, закоммитьте их и отправьте на удалённый репозиторий.

```
git add .
git commit -m "Описание изменения"
git push origin new-feature
```

5. Откройте проект в **GitLab** в браузере.
    1. Перейдите в раздел **Merge Requests** и нажмите кнопку **New merge request**.
    2. Выберите источник (source) и целевую (target) ветки.
    3. Заполните информацию о merge request
    4. Нажмите **Create merge request**.

6. Обратитесь к команде для кода-ревью.
7. После одобрения всех участников и успешного прохождения всех проверок, нажмите **Merge** для слияния изменений в целевую ветку.
